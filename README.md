# FireVault Build 0.62.0

Build 0.62.0 begins the multi-technician and cloud-sync architecture while preserving the current offline field application.

## Included
- Permanent IDs and audit metadata for sites and supported child records
- Created-by, modified-by, timestamp, device identity, and record-version fields
- Pending, synchronized, conflict, and local record states
- Automatic migration of existing local FireVault data
- New Settings → Team Sync workspace
- OneDrive / SharePoint workspace planning fields
- Conflict-policy settings and sync queue summary
- Clear notice that this build does not yet upload data
- Academy documentation updated for Build 0.62.0

Suggested commit message:

`Build 0.62.0 add multi-user and OneDrive sync-ready data foundation`
