# FireVault Build 0.57.0 — Clean Install

1. Export a current FireVault backup from **Settings → Backup**.
2. Replace the repository root files with the contents of this ZIP.
3. Commit and push the changed files.
4. Wait for GitHub Pages deployment to complete.
5. Open FireVault in Safari and confirm the green build badge reads **0.57.0**.
6. Open **Settings → Help & Manual** and confirm manual search and chapter expansion work.
7. If an older build remains cached, remove the Home Screen app, clear the FireVault website data, reopen the deployed address, verify 0.57.0, and add it to the Home Screen again.

This build uses the existing FireVault storage key, so normal upgrades preserve accounts, notes, settings, Home Layout preferences, and local field data.
