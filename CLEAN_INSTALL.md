# Deploy FireVault 0.73.1

Deploy the contents of this ZIP to the repository root without changing the GitHub Pages address. Existing FireVault data uses the same local-storage key and database format.

After deployment:

1. Open the existing FireVault Home Screen app.
2. Allow the built-in version check to find Build 0.73.1.
3. Use **Update Now** when offered, or open **Settings → Data, Sync & Support → App Updates** and choose **Check for Updates**.
4. Confirm the displayed build is 0.73.1.
5. In **Settings → Customer Import**, preview a CSV containing same-address rows with different Account IDs. Each should show as a separate New or Update record, not as an address duplicate.

Do not delete and reinstall the Home Screen app as an update method. Download an external backup first whenever a reinstall is unavoidable.
